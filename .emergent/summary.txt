<product_requirements>
RistoBrain is a multi-tenant PWA for restaurant cost control, built with React, FastAPI, and MongoDB. It features authentication, RBAC (admin/manager/waiter), multi-currency (EUR/USD, minor units), password reset, and EN/IT internationalization.

**Implemented Phases:**
*   **Phase 1:** Suppliers (CRUD, attachments), Receiving (CRUD, inventory write-through), Inventory Valuation (WAC, audit), interactive Dashboard.
*   **Phase 2:** Enhanced Ingredients (waste %, allergens, shelf life, category), Preparations (sub-recipes, computed cost/allergens, instructions), Enhanced Recipe Editor (mixed items, in-place editing, keyboard UX, live cost, allergen badges, instructions). Allergen taxonomy normalized to EU-14.
*   **Phase 3:** Sales & Wastage with WAC stock deductions, and Admin-only Users & Access management.

**Current & Future Work:**
*   **Phase 4 (Implemented, closing gaps):** Prep List (available/to make, forecast, shelf-life, manual overrides, expiry alerts, UI/UX, i18n/currency) and Order List (suggested qty, drivers, pack rounding, supplier switch, expiry alerts, RBAC).
*   **Phase 5 (Backend Complete, Frontend Implemented):** Enhanced P&L template (weekly Mon–Sun, multi-currency/locale, detailed calculations, manual lines).
*   **Phase 6 (Planned):** Suppliers+ (item mapping, price lists).
*   **Phase 7 (Planned):** RBAC Matrix (customizable permissions).
*   **Phase 8 (Planned):** Document Ingestion/OCR (parsing pipeline, review/mapping UI).
</product_requirements>

<key_technical_concepts>
-   **FastAPI**: Python web framework for the backend APIs.
-   **React**: JavaScript library for the progressive web application (PWA) frontend.
-   **MongoDB**: NoSQL database for data storage, utilizing UUIDs for IDs.
-   **JWT/RBAC**: JSON Web Tokens for authentication and Role-Based Access Control.
-   **i18n**: Internationalization using  for English and Italian.
-   **Minor Units**: Monetary value storage to prevent floating-point errors.
-   **Weighted Average Cost (WAC)**: Inventory valuation methodology.
-   **Playwright**: Frontend E2E testing framework.
-   **EU-14 Allergens**: Standardized allergen taxonomy.
</key_technical_concepts>
